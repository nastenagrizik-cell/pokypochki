# Совместные покупки

Готовый стартовый проект для GitHub + Render. Это MVP веб-сервиса для планирования совместных покупок.

## Что уже есть

- создание проектов;
- список участников;
- добавление покупок;
- назначение ответственного;
- кнопка "Взять себе";
- отметка "Куплено";
- ввод цены;
- визуализация прогресса;
- автообновление данных каждые 15 секунд.

## Структура

- `src/server.js` — backend на Express;
- `public/index.html` — интерфейс;
- `public/styles.css` — стили;
- `public/app.js` — фронтенд-логика;
- `render.yaml` — конфиг для Render;
- `package.json` — зависимости Node.js.

## Как загрузить на GitHub

1. Создай новый пустой репозиторий на GitHub.
2. Загрузи в него все файлы и папки из этого проекта, сохраняя структуру.
3. Проверь, что в корне репозитория лежат `package.json`, `render.yaml`, `README.md`, папки `public/` и `src/`.

## Как поднять в Render

1. Зайди в Render.
2. Нажми `New` → `Blueprint` или `New` → `Web Service`.
3. Выбери репозиторий.
4. Если через `Blueprint`, Render сам прочитает `render.yaml`.
5. Если через `Web Service`, укажи:
   - Build Command: `npm install`
   - Start Command: `npm start`
6. Дождись деплоя.

## Важное ограничение текущей версии

Сейчас данные хранятся в памяти сервера, чтобы тебе было легко сразу загрузить и протестировать проект. После перезапуска сервиса данные сбросятся.

Для следующей версии надо подключить PostgreSQL на Render и вынести хранение проектов, участников и покупок в базу данных.
